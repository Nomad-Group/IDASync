Output helpers

Detailed Description

All output functions use cmd structure to get information.

Parameters

Return values
0displayed as bytes
1displayed as instruction
idaman void ida_export OutBadInstruction ( void  )

Output instruction as a sequence of bytes, followed by a comment character and instruction mnemonics.

This function is used to display undocumented instructions or instructions that are improperly handled by the target assembler. OutMnem() calls this function if the current instruction is present in the array of bad instructions ( asm_t::badworks). This function outputs a colored text.

idaman bool ida_export out_one_operand ( int  n)

Use this function to output an operand of an instruction.

This function checks for the existence of a manually defined operand and will output it if it exists. Otherwise it will call processor_t::u_outop() to output operand. This function outputs a colored text.

Parameters
nnumber of operand
Return values
1operand is displayed
0operand is hidden

void out_long ( sval_t  v,
char  radix 
)

Output a number with appropriate color.

Low level function. Use OutValue() if you can. if 'voidop' is set then this function uses COLOR_VOIDOP instead of COLOR_NUMBER. 'voidop' is initialized:

  • in out_one_operand()
  • in ...cpp (before calling processor_t::d_out())
    Parameters
    vvalue to output
    radixbase (2,8,10,16)
    voidopvoidop
    • voidop==0: operand is ok
    • voidop==1: operand is void and should be output with COLOR_VOIDOP
    • voidop==2: operand can't be output as requested and should be output with COLOR_ERROR
