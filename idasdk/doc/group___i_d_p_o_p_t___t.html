#define IDPOPT_CST   6

Custom type, starting with a '{'.

Values of this type should be handled by set_options_t callbacks. E.g.,:

1 < ERROR_STRINGS =
2 < {
3 < {0, "Unknown error"},
4 < {1, "Missing filename"},
5 < {5, "Out-of-memory"}
6 < }
7 <

For values of this type, the data that will be passed as the callback's 'value' parameter is the lexer_t instance that is being used to parse the configuration file.
